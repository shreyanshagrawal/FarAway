# Commit History Guidance

## Conventional Commit Format
Use the following format for all commits:
- `feat`: A new feature or capability (e.g., `feat: add insight agent clustering logic`)
- `fix`: A bug fix (e.g., `fix: handle empty Qdrant collection on first run`)
- `chore`: Routine tasks, configuration, or build changes (e.g., `chore: add pytest configuration`)
- `docs`: Documentation updates (e.g., `docs: update README with API reference`)
- `test`: Adding or modifying tests (e.g., `test: add domain inference tests`)

## Rules for this Project
- **Never squash commits**: We want to preserve the full history of the project's development.
- **Each sub-task = one commit**: Keep commits small and focused on a single change.
- **All team members commit from their own accounts**: Do not share accounts or use generic commit authors.
- **Target**: We aim for 50+ commits by demo day to demonstrate consistent progress.
